# Registry Mirror Configuration: k3s vs k0s Comparison

## Overview

This document compares how k3s and k0s handle Docker registry mirror configuration and provides implementation guidance for k0s.

## k3s Registry Configuration

### Architecture

k3s uses a sophisticated registry configuration system:

1. **Configuration File**: `/etc/rancher/k3s/registries.yaml`
2. **Runtime Integration**: Automatically generates containerd `hosts.toml` files
3. **Dynamic Updates**: Supports runtime configuration changes
4. **Mirror Support**: Built-in support for registry mirrors and rewrites

### Key Components

#### 1. Registry Configuration Structure (k3s)

```yaml
# /etc/rancher/k3s/registries.yaml
mirrors:
  docker.io:
    endpoint:
      - "https://mirror1.example.com"
      - "https://mirror2.example.com"
  "delivery.instana.io":
    endpoint:
      - "https://delivery.instana.io"
    rewrite:
      "^rancher/(.*)": "rancher-mirror/$1"

configs:
  "delivery.instana.io":
    auth:
      username: "user@example.com"
      password: "token"
    tls:
      cert_file: "/path/to/cert.pem"
      key_file: "/path/to/key.pem"
      ca_file: "/path/to/ca.pem"
      insecure_skip_verify: false
```

**File**: `k3s/pkg/agent/containerd/config.go`

#### 3. Generated hosts.toml (k3s)

k3s generates containerd `hosts.toml` files in:
- `/var/lib/rancher/k3s/agent/etc/containerd/certs.d/{registry}/hosts.toml`

```toml
# Generated by k3s
server = "https://delivery.instana.io"

[host."https://delivery.instana.io"]
  capabilities = ["pull", "resolve"]
  [host."https://delivery.instana.io".header]
    authorization = ["Basic <base64-credentials>"]
```

## k0s Registry Configuration

### Current Architecture

k0s uses a simpler drop-in configuration approach:

1. **Drop-in Directory**: `/etc/k0s/containerd.d/`
2. **Manual Configuration**: Users create TOML files manually
3. **CRI Plugin Merging**: k0s merges CRI plugin configs automatically
4. **Modern Format**: Uses containerd's native `hosts.toml` format

### Key Components

#### 1. k0s Configuration Structure

**File**: `k0s/pkg/component/worker/containerd/component.go`

```go
func (c *Component) setupConfig() error {
    // Check if config is k0s managed
    k0sManaged, err := isK0sManagedConfig(c.confPath)
    if err != nil {
        return err
    }
    
    if !k0sManaged {
        logrus.Infof("containerd config file %s is not k0s managed, skipping config generation", c.confPath)
        return nil
    }
    
    // Create configurer to handle imports
    configurer := &configurer{
        loadPath:   filepath.Join(c.importsPath, "*.toml"),
        pauseImage: c.Profile.PauseImage.URI(),
        log:        logrus.WithField("component", "containerd"),
    }
    
    // Handle imports and merge CRI configs
    config, err := configurer.handleImports()
    if err != nil {
        return fmt.Errorf("can't handle imports: %w", err)
    }
    
    // Write final configuration
    return c.writeConfig(config)
}
```

#### 2. Drop-in Configuration (k0s)

Users manually create:

```bash

[host."https://delivery.instana.io"]

  [host."https://delivery.instana.io".header]
    authorization = ["Basic <base64-credentials>"]
```

## Comparison Table

| Feature | k3s | k0s |
|---------|-----|-----|
| **Config File** | `/etc/rancher/k3s/registries.yaml` | Manual TOML files in `/etc/k0s/containerd.d/` |
| **Format** | YAML (k3s-specific) | TOML (containerd native) |
| **Mirror Support** | Built-in with `mirrors` section | Manual configuration |
| **Auth Config** | Centralized in `configs` section | Per-registry `hosts.toml` |
| **Rewrite Rules** | Supported via `rewrite` | Not directly supported |
| **Dynamic Updates** | Automatic via file watch | Requires k0s restart |
| **Code Generation** | Automatic `hosts.toml` generation | Manual TOML creation |
| **Complexity** | Higher (custom parser) | Lower (native containerd) |
| **Flexibility** | Opinionated structure | Full containerd flexibility |

## Implementation Recommendations for k0s

### Option 1: Add k3s-like Registry Configuration (Recommended)

Create a similar YAML-based configuration system for k0s:

```go
// k0s/pkg/component/worker/containerd/registry.go

package containerd

import (
    "encoding/base64"
    "fmt"
    "net/url"
    "os"
    "path/filepath"
    
    "gopkg.in/yaml.v2"
)

// RegistryConfig represents the k0s registry configuration
type RegistryConfig struct {
    Mirrors map[string]MirrorConfig `yaml:"mirrors"`
    Configs map[string]HostConfig   `yaml:"configs"`
}

type MirrorConfig struct {
    Endpoint []string          `yaml:"endpoint"`
    Rewrite  map[string]string `yaml:"rewrite,omitempty"`
}

type HostConfig struct {
    Auth *AuthConfig `yaml:"auth,omitempty"`
    TLS  *TLSConfig  `yaml:"tls,omitempty"`
}

type AuthConfig struct {
    Username string `yaml:"username,omitempty"`
    Password string `yaml:"password,omitempty"`
    Auth     string `yaml:"auth,omitempty"`
}

type TLSConfig struct {
    CertFile           string `yaml:"cert_file,omitempty"`
    KeyFile            string `yaml:"key_file,omitempty"`
    CAFile             string `yaml:"ca_file,omitempty"`
    InsecureSkipVerify bool   `yaml:"insecure_skip_verify,omitempty"`
}

// LoadRegistryConfig loads registry configuration from YAML file
func LoadRegistryConfig(path string) (*RegistryConfig, error) {
    data, err := os.ReadFile(path)
    if err != nil {
        if os.IsNotExist(err) {
            return &RegistryConfig{}, nil
        }
        return nil, err
    }
    
    var config RegistryConfig
    if err := yaml.Unmarshal(data, &config); err != nil {
        return nil, fmt.Errorf("failed to parse registry config: %w", err)
    }
    
    return &config, nil
}

// GenerateHostsToml generates containerd hosts.toml content
func (r *RegistryConfig) GenerateHostsToml(registry string) (string, error) {
    var content string
    
    // Get mirror config
    mirror, hasMirror := r.Mirrors[registry]
    config, hasConfig := r.Configs[registry]
    
    if !hasMirror && !hasConfig {
        return "", fmt.Errorf("no configuration found for registry %s", registry)
    }
    
    // Determine server URL
    serverURL := fmt.Sprintf("https://%s", registry)
    if hasMirror && len(mirror.Endpoint) > 0 {
        serverURL = mirror.Endpoint[0]
    }
    
    content += fmt.Sprintf("server = %q\n\n", serverURL)
    
    // Add mirror endpoints
    if hasMirror {
        for _, endpoint := range mirror.Endpoint {
            content += r.generateHostSection(endpoint, &config)
        }
    } else {
        content += r.generateHostSection(serverURL, &config)
    }
    
    return content, nil
}

func (r *RegistryConfig) generateHostSection(endpoint string, config *HostConfig) string {
    var section string
    
    section += fmt.Sprintf("[host.%q]\n", endpoint)
    section += "  capabilities = [\"pull\", \"resolve\"]\n"
    
    // Add authentication
    if config != nil && config.Auth != nil {
        section += fmt.Sprintf("  [host.%q.header]\n", endpoint)
        
        var authToken string
        if config.Auth.Auth != "" {
            authToken = config.Auth.Auth
        } else if config.Auth.Username != "" && config.Auth.Password != "" {
            creds := fmt.Sprintf("%s:%s", config.Auth.Username, config.Auth.Password)
            authToken = base64.StdEncoding.EncodeToString([]byte(creds))
        }
        
        if authToken != "" {
            section += fmt.Sprintf("    authorization = [\"Basic %s\"]\n", authToken)
        }
    }
    
    // Add TLS configuration
    if config != nil && config.TLS != nil {
        if config.TLS.CAFile != "" {
            section += fmt.Sprintf("  ca = %q\n", config.TLS.CAFile)
        }
        if config.TLS.CertFile != "" {
            section += fmt.Sprintf("  client = [[%q, %q]]\n", config.TLS.CertFile, config.TLS.KeyFile)
        }
        if config.TLS.InsecureSkipVerify {
            section += "  skip_verify = true\n"
        }
    }
    
    section += "\n"
    return section
}

// ApplyRegistryConfig applies registry configuration to containerd
func (c *Component) ApplyRegistryConfig(registryConfigPath string) error {
    config, err := LoadRegistryConfig(registryConfigPath)
    if err != nil {
        return err
    }
    
    certsDir := filepath.Join(c.importsPath, "certs.d")
    if err := os.MkdirAll(certsDir, 0755); err != nil {
        return err
    }
    
    // Generate hosts.toml for each configured registry
    allRegistries := make(map[string]bool)
    for registry := range config.Mirrors {
        allRegistries[registry] = true
    }
    for registry := range config.Configs {
        allRegistries[registry] = true
    }
    
    for registry := range allRegistries {
        hostsToml, err := config.GenerateHostsToml(registry)
        if err != nil {
            logrus.Warnf("Failed to generate hosts.toml for %s: %v", registry, err)
            continue
        }
        
        registryDir := filepath.Join(certsDir, registry)
        if err := os.MkdirAll(registryDir, 0755); err != nil {
            return err
        }
        
        hostsFile := filepath.Join(registryDir, "hosts.toml")
        if err := os.WriteFile(hostsFile, []byte(hostsToml), 0644); err != nil {
            return err
        }
        
        logrus.Infof("Generated hosts.toml for registry %s", registry)
    }
    
    return nil
}
```

### Option 2: Enhance Current Drop-in System

Improve the existing drop-in system with helper utilities:

```go
// k0s/pkg/component/worker/containerd/registry_helper.go

package containerd

import (
    "encoding/base64"
    "fmt"
    "os"
    "path/filepath"
)

// RegistryHelper provides utilities for registry configuration
type RegistryHelper struct {
    certsDir string
}

func NewRegistryHelper(certsDir string) *RegistryHelper {
    return &RegistryHelper{certsDir: certsDir}
}

// AddRegistry adds a registry with authentication
func (h *RegistryHelper) AddRegistry(registry, username, password string, insecure bool) error {
    registryDir := filepath.Join(h.certsDir, registry)
    if err := os.MkdirAll(registryDir, 0755); err != nil {
        return err
    }
    
    authToken := base64.StdEncoding.EncodeToString([]byte(fmt.Sprintf("%s:%s", username, password)))
    
    hostsToml := fmt.Sprintf(`server = "https://%s"

[host."https://%s"]
  capabilities = ["pull", "resolve"]
`, registry, registry)
    
    if username != "" && password != "" {
        hostsToml += fmt.Sprintf(`  [host."https://%s".header]
    authorization = ["Basic %s"]
`, registry, authToken)
    }
    
    if insecure {
        hostsToml += "  skip_verify = true\n"
    }
    
    hostsFile := filepath.Join(registryDir, "hosts.toml")
    return os.WriteFile(hostsFile, []byte(hostsToml), 0644)
}

// AddMirror adds a mirror for a registry
func (h *RegistryHelper) AddMirror(registry string, mirrors []string) error {
    registryDir := filepath.Join(h.certsDir, registry)
    if err := os.MkdirAll(registryDir, 0755); err != nil {
        return err
    }
    
    hostsToml := fmt.Sprintf(`server = "https://%s"

`, registry)
    
    for _, mirror := range mirrors {
        hostsToml += fmt.Sprintf(`[host."%s"]
  capabilities = ["pull", "resolve"]

`, mirror)
    }
    
    hostsFile := filepath.Join(registryDir, "hosts.toml")
    return os.WriteFile(hostsFile, []byte(hostsToml), 0644)
}
```

## Recommendation

**Implement Option 1** (k3s-like configuration) because:

1. ✅ **User-Friendly**: YAML is easier than manual TOML
2. ✅ **Centralized**: Single file for all registries
3. ✅ **Consistent**: Similar to k3s, easier migration
4. ✅ **Automated**: Generates hosts.toml automatically
5. ✅ **Maintainable**: Easier to update and version control

## Migration Path

1. **Phase 1**: Implement registry config parser
2. **Phase 2**: Add hosts.toml generator
3. **Phase 3**: Integrate with component startup
4. **Phase 4**: Add CLI commands for registry management
5. **Phase 5**: Document and release

## Conclusion

While k0s's current approach is simpler and uses containerd's native format, implementing a k3s-like registry configuration system would provide better user experience and easier management, especially for users migrating from k3s or managing multiple registries.
